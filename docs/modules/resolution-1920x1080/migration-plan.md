# 1920x1080 迁移计划

## 目标

把运行时逻辑画布迁移到 `1920x1080`，同时在替换资源准备好之前，尽量保持旧 `800x600` UI 素材稳定可用。

## 文件影响范围

### 第一阶段候选

- `system/Status.tjs`
  - 修改逻辑根常量。
  - 目标值：
    - `WINDOW_WIDTH = 1920`
    - `WINDOW_HEIGHT = 1080`
    - `WINDOW_CENTER_X = 960`
    - `WINDOW_CENTER_Y = 540`
- `system/Window.tjs`
  - 验证窗口和全屏缩放中没有残留的 4:3 假设。
  - 缩放数学继续基于常量。
- `system/ADVObject.tjs`
  - 全画布 ADV 对象和色块填充通过常量扩展。
- `system/ADVScreen.tjs`
  - 根画布、转场精灵、背景兜底填充、截图复制、选项底层通过常量扩展。
  - 默认图片中心 `400,300` 只有在视觉冒烟测试确认语义为“屏幕中心”后，才替换为 `WINDOW_CENTER_X, WINDOW_CENTER_Y`。
- `system/ConfigWindow.tjs`
  - 叠层和提示框边界跟随全窗口常量。
  - 新的 16:9 画面尺寸选项，需要 UI 组提供对应文案和按钮资源后再改。

### 等 UI/图片资源完成后再处理

- `system/MessageFrame.tjs`
  - 对话框尺寸、文字边距、按钮坐标、隐藏/关闭/闪烁提示位置、系统菜单点击层。
- `system/SystemWindow.tjs`
  - 存档/读档/历史底图、预览位置、历史文本区域、存档格素材。
- `system/Album.tjs`
  - 底部查看菜单高度和位置、预览横竖滑条、CG 缩略图布局。
- `system/Title.tjs`
  - Logo/注意事项位置、标题按钮条、标题菜单位置。
- `system/Movie.tjs`
  - 视频层尺寸策略取决于视频素材是否重制。

### 第一阶段不直接处理

- `system/MessageArea.tjs`
  - 它从调用方接收尺寸和边距。只有新对话 UI 暴露换行或排版问题时再改。

## 推荐修改顺序

1. 基线截图
   - 在当前 `800x600` 下运行游戏。
   - 截图标题、ADV 对话、选项、设置、存档/读档、历史、CG 查看、视频播放。
   - 保存为变更前证据。

2. 核心常量
   - 只修改 `system/Status.tjs` 的逻辑常量。
   - 立即启动测试。
   - 旧 UI 素材出现在 `1920x1080` 画布的一角或旧位置是可接受的阶段性结果。

3. 窗口缩放
   - 测试 `system/Window.tjs`：默认窗口、手动缩放、进入/退出全屏、拖拽窗口边缘。
   - 如果出现比例漂移，先修 `Window.tjs`，不要继续动 UI。

4. ADV 全画布行为
   - 验证 `ADVScreen.tjs` 和 `ADVObject.tjs` 的全屏填充、转场、截图是否覆盖 `1920x1080`。
   - 默认中心 `400,300` 等确认语义后再替换。

5. 存档缩略图和历史预览策略
   - 决定存档缩略图保存完整 `1920x1080` 截图，还是保存缩小后的固定尺寸。
   - 如果格式期望 `800x600`，应显式缩放缩略图，而不是直接复制全窗口。

6. 视频策略
   - 视频仍是 `800x600` 时，居中显示、整数放大、等比适配或加设计框需明确。
   - 视频已重制时，再把 `Movie.tjs` 改为 `1920x1080` 或使用适配辅助函数。

7. UI 资源集成
   - UI/图片资源交付后，再改 `MessageFrame.tjs`、`SystemWindow.tjs`、`ConfigWindow.tjs`、`Album.tjs`、`Title.tjs`。
   - 可见素材和点击区域必须同批调整。

8. 最终清扫
   - 重新运行数字扫描。
   - 所有剩余 `800`、`600`、`400`、`300` 都必须标注为：时间、资源坐标、旧预设或未解决迁移项。

## 风险

- 宽高比风险：4:3 改 16:9 后，旧 CG 定位可能露出空边或错误裁切。
- UI 不匹配风险：可见按钮和点击区域是分开写的，只移动一边会导致点击错位。
- 存档风险：缩略图或自动存档截图变大后，可能影响存档性能和预览布局。
- 视频风险：`800x600` 视频居中后可能太小，拉伸又会变形。
- 文字风险：对话和历史边距可能过宽或过窄。
- 部分 HD 常量风险：现有 `1280x720` 值不自动等于 `1920x1080` 目标。

## 验证方法

### 静态验证

运行：

```powershell
rg -n "WINDOW_WIDTH|WINDOW_HEIGHT|\b800\b|\b600\b|\b400\b|\b300\b|\b1280\b|\b720\b" system/Status.tjs system/Window.tjs system/ADVScreen.tjs system/ADVObject.tjs system/MessageFrame.tjs system/MessageArea.tjs system/Movie.tjs system/SystemWindow.tjs system/ConfigWindow.tjs system/Album.tjs system/Title.tjs
```

确认每个剩余字面量都有归属：

- 逻辑分辨率
- UI 坐标或资源尺寸
- 图片中心
- 视频区域
- 存档缩略图
- CG 浏览
- 鼠标点击区
- 动画时间

### 运行时验证

- 启动和标题：
  - 窗口打开为 `1920x1080`。
  - 标题背景和菜单不崩溃。
  - Logo/注意事项流程仍可见。
- 窗口：
  - 窗口缩放保持 16:9。
  - 全屏进入/退出后窗口仍可操作。
- ADV：
  - 背景填满或按策略留边。
  - 对话框可显示/隐藏。
  - 选项按钮出现并可点击。
  - 截图覆盖预期视口。
- 设置：
  - 画面模式和尺寸设置可打开。
  - 提示弹窗在屏幕内。
- 存档/读档/历史：
  - 缩略图可生成。
  - 页面可打开。
  - 历史预览不超出边界。
- Album：
  - CG 查看可打开。
  - CG 平移/裁切滑条不反向、不溢出。
- 视频：
  - 按既定策略播放：源尺寸居中、适配或重制 `1920x1080`。

## 本模块完成标准

- 核心分辨率常量为 `1920x1080`。
- 全画布层使用新常量。
- UI/资源相关坐标已经匹配新素材，或被明确记录为阶段性旧坐标。
- 已分配文件中没有未分类的 `800`、`600`、`400`、`300`。
- 标题、ADV、设置、存档/读档/历史、Album、视频流程冒烟测试通过。
