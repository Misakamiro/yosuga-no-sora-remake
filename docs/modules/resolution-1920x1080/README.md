# 1920x1080 分辨率模块交接

## 目标

将游戏核心分辨率从 `800x600` 迁移到 `1920x1080`，但不能简单替换所有数字。当前脚本里混合了三类内容：

- 逻辑画布尺寸：`WINDOW_WIDTH`、`WINDOW_HEIGHT`、中心点和宽高比等派生值。
- 旧 `800x600` UI 和素材坐标：按钮行、对话框、标题 Logo、设置/存档/读档/历史界面。
- 运行时视口计算：全屏缩放、CG 浏览裁切范围、截图复制尺寸、视频居中、点击区域。

迁移时必须把这些概念分开。第一阶段只让引擎画布和明确依赖常量的计算具备 `1920x1080` 能力；旧 UI 资源坐标在 UI 和图片资源完成前暂时保持不动。

- `system/MessageFrame.tjs`

本轮没有修改图片资源、剧本脚本、封包文件，也没有进行批量数字替换。

## 推荐方向

采用兼容优先的迁移方式：

1. 通过 `Status.tjs` 把逻辑画布改为 `1920x1080`。
2. 确认 `Window.tjs` 的窗口缩放和全屏逻辑仍然基于 `WINDOW_WIDTH`、`WINDOW_HEIGHT`、`WINDOW_ASPECT`。
3. 只更新已经明确使用逻辑常量或明显派生中心点的路径。
4. UI 界面和点击区域等待 UI 组给出 `1920x1080` 布局、图片组给出匹配素材后再移动。
5. 只有在语义归属明确时，才替换硬编码的 `800x600` 默认值，例如视频区域、默认图片中心、存档缩略图源尺寸、CG 裁切视口或具体 UI 面板。

## 关键风险

- `800`、`600`、`400`、`300` 不一定都是分辨率值，很多 `300` 是动画时长。
- 部分文件已经存在 `1280x720` 或 `1280x128` 等值，可能是早期 HD 尝试，不能当作目标值。
- `800x600` 到 `1920x1080` 会从 4:3 改为 16:9，中心点和素材锚点需要重新设计，不能只做缩放。
- 现有 CG 和背景素材可能仍是 `800x600`，代码必须明确是放大、裁切、留边，还是要求重制素材。

## 最小验证项

实施后至少验证：

- 能以 `1920x1080` 窗口启动到标题。
- 窗口/全屏切换和手动缩放保持 16:9。
- ADV 场景可进入，对话框、右键菜单、选项、路线提示可显示并可点击。
- 设置、存档/读档、历史、CG 模式、音乐、回忆、标题菜单流程可打开。
- 视频场景按既定策略居中或缩放。
- 存档缩略图创建和显示无偏移、无裁切错误。
- UI 素材替换后，点击区域与可见按钮一致。
