## 关键依赖

1. `engine-compat` 是所有代码模块前置条件。启动链路未稳定前，不允许把 UI 或剧本问题判定为最终问题。
2. `resolution-1920x1080` 是 `ui-redesign` 和 `ui-animation` 的坐标基础。坐标未冻结前，UI 资源只能作为候选图。
3. `scenario-parser` 必须先提供图像引用和选项清单，`image-remaster` 才能按优先级处理剧情关键图。
4. `image-remaster` 输出必须经过 `image-qa`，再进入 `ui-redesign` 或 packaging 候选包。
5. `ui-animation` 必须经过 `performance` 联调，不能单独宣布通过。
6. `packaging` 必须等待 engine、scenario、resolution、UI、image、performance 都有候选验收记录。

## 每周验收节奏

### 每周一

- 每个负责人提交上周进度报告。
- 项目总控更新阻塞项和本周目标。
- 确认跨模块交接单是否已被接收。

### 每周三

- 检查本周高风险任务。
- 对新增跨模块修改做边界确认。
- 对图像和 UI 批次做中途抽查。

### 每周五

- 执行本周验收。
- 所有人提交工作日志。
- 将通过项、返修项、阻塞项分开记录。

## 阶段验收清单

### P1 出口

- `startup.tjs` 启动链路说明完成。
- `system\` 兼容文件清单完成。
- `scenario\*.ks` 和 `content-data\*.ks` 清单完成。
- 图像资源按背景、CG、UI、缩略图分组完成。

### P2 出口

- 主窗口目标为 `1920x1080`。
- ADV 主画面截图覆盖背景、立绘、消息框、选项。
- 旧 4:3 图像占位策略明确。

### P3 出口

- 标题、配置、消息框、选项、存读档、CG 页面有新版截图。
- 第一批图像通过 image-qa。
- UI 页面脚本路径和资源路径映射完成。

### P4 出口

- 动画触发和清理逻辑清楚。
- 启动、页面切换、大图加载有性能记录。
- 低性能降级策略有明确开关或规则。

### P5 出口

- 便携版和安装包候选版生成。
- 包内不包含 `savedata\`、临时目录、未验收资源。
- 新目录安装后能启动并进入标题。

### P6 出口

- 最终安装包、便携包、SHA256 文件齐全。
- 安装、启动、配置、存读档、退出、卸载全部通过。
- 发布说明写明版本、已知问题和运行要求。

## 延期处理规则

- 单模块延期 1-2 天：模块负责人更新进度报告，说明影响范围。
- 跨模块阻塞超过 2 天：项目总控召开边界确认，调整依赖顺序。
- 候选包前出现阻塞级 bug：暂停 packaging，先回到责任模块修复。
- 图像批次未通过 QA：不得进入最终包，可使用旧图或占位策略，但必须在已知问题中写明。
