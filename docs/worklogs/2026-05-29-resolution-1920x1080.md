## 发现

- `system/Status.tjs` 是逻辑尺寸根节点：`WINDOW_WIDTH = 800`、`WINDOW_HEIGHT = 600`。
- 改 `Status.tjs` 后，`system/Window.tjs` 应作为第一运行验证目标。
- `system/Movie.tjs` 将视频播放层硬编码为 `800x600`，再在逻辑窗口内居中。
- `system/ADVScreen.tjs` 同时包含可通过常量扩展的画布逻辑，以及不安全旧中心/旧位置值：
  - 默认图片中心 `400,300`。
  - 选项垂直中心 `300`。
  - 角色自动站位偏移 `-400`、`-300`、`300`、`400`。
- `system/MessageFrame.tjs` 与旧 UI 素材和点击位置强绑定。
- `system/Album.tjs` 混合了全窗口 CG 数学、旧底部菜单和旧缩略图布局。
- `system/SystemWindow.tjs` 的叠层尺寸使用常量，但预览/历史数学需要视觉确认。
- `system/ConfigWindow.tjs` 包含旧 4:3 画面尺寸预设和部分 `1280x720` 全局底图。
- `system/Title.tjs` 包含旧 Logo/标题菜单位置。
- `system/MessageArea.tjs` 本轮没有直接命中被扫描的分辨率值。
