# UI 素材需求清单

## 交付格式

图片组需要交付高分辨率源文件，以及 KRKRZ 可直接使用的运行时导出文件。运行时文件名可以在脚本确认后替换现有 `FRM_*`，也可以先增加为 `FRM_HD_*`，便于渐进迁移。

每个素材必须附带元数据：

- 运行时文件名
- 像素尺寸
- 图案/帧数量
- 状态帧顺序
- 背景是否透明
- 适用页面和对应代码文件

## 字体需求

当前 `font` 目录包含日文字体 `.tft` 尺寸梯度，例如 Sura、New Cinema、Humming、Rodin Cattleya、Rodin Happy、Tsukushi Mincho。1080p 下至少保留 `24`、`30`、`36`、`48` 尺寸。若中文可读性是重点，需要提供兼容中日文字形的字体集，并在替换 `.tft` 前验证字形覆盖。

## 需要实现组决定的问题

- 覆盖旧 `FRM_*` 名称，还是引入 `FRM_HD_*` 并逐步改脚本。
- 存档/读档网格使用 `4x2` 保证可读性，还是使用 `4x3` 更接近当前每页 9 格。
- 标题背景由图片组重新构图，还是基于现有标题图裁切/扩展。
- `thumb/*` CG 缩略图从最终 CG 重新生成，还是用更大卡片遮罩显示现有缩略图。
