## 类型判断

| 类型 | 主要路径 | 代表文件 | 当前尺寸特征 | 处理目标 |
| --- | --- | --- | --- | --- |
| 背景 | `bg`, `content-data/bg` | `bg/B01a.tlg` | 常见 `800x600`，另有 `800x1200`、`800x1201`、`1400x600`、`1708x600`、`1766x600` | 16:9 扩图到 `1920x1080` |
| CG / event | `content-data/char` | `content-data/char/SP01.tlg`, `content-data/char/ca01_01l.tlg` | 常见 `800x600`、`901x640`、`812x639` 等 | 按画面主体扩图到 `1920x1080` |
| 立绘 / 人物层 | `content-data/char` | `content-data/char/cc01_01l.tlg` | 常见竖向或半身尺寸，例如 `714x738`、`435x610`、`565x622` | 先保留透明 / 位置语义，是否上 16:9 画布需按调用方式确认 |
| UI 图 | `frame`, `content-data/frame` | `frame/FRM_02A02.tlg` | 小组件到全屏 UI 混合，例如 `576x23`、`306x230`、`640x568`、`800x600` | 按 UI 组需求，不默认统一到 `1920x1080` |
| 缩略图 | `thumb`, `content-data/thumb` | `content-data/thumb/ca01_01TS.tlg` | `100x75` 为主，另有 `160x125`、`160x124`、`194x189` | 按系统窗口需求，通常由最终 CG 重新生成 |
| 转场图 | `rule`, `content-data/rule` | `rule/CLOUD_A.png` | 全部 `800x600` PNG | 按引擎规则确认是否扩到 `1920x1080` |

### 注意事项

- `bg/B01a.png` 与 `bg_output/B01a.png` 均为 0 字节占位，不得作为可处理源图。
- `frame` / `content-data/frame` 中发现 1 个 TLG5 文件尺寸头解析结果异常，后续实际处理前必须用 TLG 解码器复核，不要只信头部快读结果。
- `content-data/char` 是混合目录，不能按目录名简单等同于立绘；处理前必须结合文件名前缀、尺寸、透明信息和调用位置复核类型。
