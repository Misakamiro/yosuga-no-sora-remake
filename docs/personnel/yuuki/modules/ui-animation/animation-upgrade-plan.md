# 动画升级计划

## 设计目标

在保留现有 KRKRZ 动画模型的前提下，让 UI 动画具备原生 `1920x1080` 的观感。目标是顺滑、清晰、低风险：短淡入、小幅方向位移、明确焦点变化，普通 UI 交互不使用沉重的全屏特效。

在 `1920x1080` 布局和 UI 素材足够稳定、能够验证真实位置前，不要实施本计划。

## 共享动画规则

使用现有 `Sprite` API：

- 淡入淡出：`setBlendingEnvelope()` + `beginActivation()`。
- 位移：`setMovingEnvelope()` + `beginActivation()`。
- 淡入淡出加位移：先设置两个 envelope，再调用 `beginActivation()`。
- 大状态切换：只在转场进行中临时禁用点击区域，并从一个完成回调恢复。

建议时长：

| 动画类型 | 时长 | 说明 |
| --- | --- | --- |
| 悬停/焦点细节 | `80-120ms` | 只在按钮需要时使用，避免鼠标移动时对每个按钮做逐帧工作。 |
| 小面板/菜单出现 | `150-220ms` | 系统菜单、子菜单、快速叠层。 |
| 对话框显示/隐藏 | `180-260ms` | 阅读时必须及时；跳过模式应降为 `0ms`。 |
| 选项列表出现 | `180-260ms` | 可按项错开 `20-35ms`，但要谨慎。 |
| 设置/存档/读档窗口切换 | `220-320ms` | 使用单一分组转场，不要给大量子控件分别动画。 |
| 标题场景入场/退场 | 入场 `500-1000ms`，退场 `500-1500ms` | 标题可比 ADV UI 慢，但避免长时间锁输入。 |

推荐缓动：

- 现有代码已用加速度 `2` 的滑入/滑出可继续使用。
- 透明度默认线性，除非特定场景需要更柔和曲线。
- 普通 UI 避免缩放/旋转，旧低分辨率素材会更容易暴露瑕疵。

## 标题菜单

当前行为：

- 标题遮罩 `1000ms` 淡出。
- 主菜单 `500ms` 淡入。
- Bonus 菜单交叉淡入淡出并移动 `32px`。
- 开始游戏时标题淡出，部分精灵移动。

`1920x1080` 建议：

1. 标题背景和 Logo 先出现，然后菜单作为一组入场。
2. 菜单组从最终位置下方 `24-36px` 开始，`450-600ms` 内从透明到不透明。
3. 标题按钮悬停保持图片状态切换，后续如果素材支持再加轻微透明度/位置细节。
4. Bonus/图库子菜单保留交叉淡入思路，但 1080p 滑动距离建议约 `42-56px`，不要直接用 `32px`。
5. 开始/退出场景时，整个标题组 `600-900ms` 淡出；只有装饰性标题精灵可漂移，避免大面积全屏层移动。

涉及文件：

- `system/Title.tjs`
- 只有未来需要辅助常量时才考虑 `system/Sprite.tjs`

实现注意：

- 旧标题菜单坐标必须等 UI 布局给出最终 `1920x1080` 位置后再替换。
- `SPR_COVER` 淡出继续作为全场景转场。
- 不要把 800 下所有标题退出偏移机械乘比例，应该按视觉距离重调。

## 对话框显示 / 隐藏

当前行为：

- `MessageFrame.show()` 和 `hide()` 淡入淡出整个消息框。
- `showMessage()` 和 `hideMessage()` 淡入淡出 `_base`。
- `setType()` 在不同消息框样式之间交叉淡入。
- 系统菜单区域和消息底板可以独立移动。

`1920x1080` 建议：

1. 默认 ADV 对话框显示：透明度 `0 -> 255`，从最终 Y 下方 `+28px` 滑入，耗时 `180-240ms`。
2. 隐藏：透明度 `255 -> 0`，向下 `+18-24px`，耗时 `120-180ms`。
3. 对话框类型切换保留交叉淡入，在布局稳定前避免新旧面板同时移动。
4. 光标闪烁保留现有淡入行为；位置应在消息布局更新后从文本光标推导。
5. 跳过模式或 `CONFIG.screenEffect == 0` 时，对话框显示/隐藏动画降为 `0ms`。

涉及文件：

- `system/MessageFrame.tjs`
- `system/Sprite.tjs`
- 如需要处理点击区域状态，可涉及 `system/SelectItem.tjs`

实现注意：

- 当前对话框常量仍是旧坐标，必须等 `FRAME_WIDTH`、`FRAME_HEIGHT`、按钮位置、消息区域更新后再调动画。
- 保持 `show_end` 和 `hide_end` 回调不变。

## 系统菜单

当前行为：

- `SystemMenuArea.enter()` 滑到 `0,0`，`leave()` 滑到 `0,48`，耗时 `150ms`。
- `SystemMenu` 是 `800x33`。
- 子菜单立即显示/隐藏。

`1920x1080` 建议：

1. 保留紧凑系统菜单条，但锚定到 UI 布局计划中的 1080p 安全区。
2. 锁定/解锁菜单显示时使用 `48-64px` 短滑动，并可从透明度 `180 -> 255`。
3. 子菜单从连接方向偏移 `8-16px`，`120-160ms` 内淡入并滑入。
4. 隐藏子菜单可立即完成或 `80-120ms`；不可让不可见菜单继续接收输入。
5. 保留 `SubMenu.show()` 的屏幕边界限制，等最终位置明确后再加动画。

涉及文件：

- `system/MessageFrame.tjs`
- `system/SelectItem.tjs`

实现注意：

- `SubMenu.show()` 当前先算最终位置再 `visible = true`。低风险改法是先算最终位置，根据连接方向临时偏移，再动画回最终位置。
- 不要给每个子菜单新增独立定时器，优先复用 `Sprite` activation。

## 选项 / 选择列表出现

当前行为：

- `SelectItem.tjs` 提供按钮、滑条、单选、子菜单、链式导航基础控件。
- 选项具体布局在其他调用处，基础控件多数是立即状态切换。

`1920x1080` 建议：

1. 选项作为一组出现在 `1920x1080` 安全区中部。
2. 每个选项从最终位置下方 `+18-24px` 淡入滑入。
3. 可选错开：每项 `20-35ms`，总时长上限 `320ms`。
4. 悬停/焦点默认继续即时切换图片状态，等素材支持后再加动画。
5. 键盘焦点不要依赖动画完成；如果不用转场，可立即启用输入。

涉及文件：

- `system/SelectItem.tjs`
- 实际选项位置可能由外部调用方负责，不要在本模块改解析器。

实现注意：

- 选项动画应放在 UI 层/调用层，不要全局改所有 `Button`。全局按钮动画会影响设置、存档、读档，并增加交互延迟风险。

## 设置 / 存档 / 读档窗口切换

当前行为：

- 这些窗口不在本模块直接可编辑清单中，但标题和系统菜单会打开它们。
- 共享控件来自 `SelectItem.tjs`。

`1920x1080` 建议：

1. 每个窗口作为一个大表面/组处理：淡入加小幅垂直或水平位移。
2. 设置标签页内容可交叉淡入或滑动 `12-20px`，避免给每个滑条和开关单独动画。
3. 存档/读档翻页时，卡片网格可淡入或按一行/一列滑动，耗时 `180-240ms`。
4. 从标题或 ADV 打开时，保持一致模态转场，约 `220-320ms`。
5. 返回标题/ADV 前，必须先恢复点击区域和可见状态。

涉及文件：

- `system/SelectItem.tjs` 用于共享控件。
- `system/Title.tjs` 和 `system/MessageFrame.tjs` 作为入口。
- 未来可能需要 `system/ConfigWindow.tjs` 和 `system/SystemWindow.tjs`，这些不在本模块当前编辑范围内，实施前需另行分配。

当前资源：

- `rule/curtain_open.png`、`rule/curtain_close.png`
- `rule/shutter_open.png`、`rule/shutter_close.png`
- `rule/WIP_*`
- `rule/concentrate*`
- `rule/MOZ*`

`1920x1080` 建议：

- 规则遮罩只用于全屏场景转场。
- 如果视觉上适合新 16:9 素材，标题到场景、场景到标题可用 `curtain` / `shutter`。
- 不要把重型规则转场用于对话框、悬停、子菜单、设置标签页、存档/读档卡片切换。
- 每张规则图都要在 `1920x1080` 下验证。如果遮罩分辨率低，应重做遮罩，或只保留给旧场景转场。

## 实施阶段

1. 文档和映射：本轮已完成。
2. 布局依赖：等待 `1920x1080` UI 布局和素材到位。
3. 在最小合适文件中引入动画常量或辅助函数。
4. 使用现有基础控件更新标题、菜单、对话框动画。
5. 核心 UI 可用后再增加子菜单和选项组动画。
6. 使用屏幕效果开/关、跳过模式、键盘导航、鼠标悬停、快速打开/关闭进行验证。

## 禁止事项

- 不要在未评估性能和视觉效果前，全局给每个 `Button.onMouseEnter()` / `onMouseLeave()` 加动画。
- 不要在已有 `Sprite.beginActivation()` 可用时新增帧循环/定时器系统。
- 不要对大面积全屏表面做缩放或旋转。
- 不要把所有 `16`、`32`、`48`、`64` 位移值机械乘 `2.4`，应按用途调。
- 本模块不要改剧本解析器、引擎启动、原始图片资源或发布包。
