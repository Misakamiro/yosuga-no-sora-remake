# 图像通过标准

## 通用验收

一张图只有同时满足以下条件，才能标记为“已通过”：

- 文件可打开，非 0 字节。
- 文件尺寸与目标尺寸一致，或有明确的 UI / 立绘审批尺寸记录。
- 没有覆盖、移动或删除原始资源。
- 交付路径、处理阶段、工具参数和 QA 结论完整记录。
- 肉眼检查无明显黑边、拉伸、锯齿、糊边、穿帮或色调断层。

## 背景 / CG / event

目标默认是 `1920x1080`，除非另有审批记录。

必须通过：

- 画面主体没有被 NovelAI 改写。
- 没有新增无关人物、文字、水印、异物或剧情外物件。
- 左右扩图区域与原图透视、光照、纹理方向一致。
- 接缝不可见；不得有重复纹理、断裂线、硬边或涂抹感。
- 人脸、眼睛、手部、头发和衣物结构不能变形。
- 色调、颗粒、锐度和原图区域一致。
- 16:9 构图保留原叙事重点，不能裁掉关键人物、脸、手或道具。

返工判断：

- 主体被改写：需重跑 NovelAI。
- 透视或大面积背景生成错误：需重跑 NovelAI。
- 小范围接缝、色调、穿帮：需 PS 修复。
- 放大后过锐、糊边、噪点或线条破碎：需重跑 waifu2x。

## 立绘 / 人物层

立绘不默认强制扩成 16:9。必须先确认引擎调用方式和 UI 摆放方式。

必须通过：

- alpha 通道完整，透明边缘干净。
- 头顶、发丝、脚底、衣物边缘没有被截断。
- 人物比例、脸型、表情、站姿不被改变。
- 线稿和上色边缘没有明显糊边、黑边、白边或锯齿。
- 如果扩画布，扩展区域不得改变站位语义。

## UI 图

UI 图不默认强制扩成 `1920x1080`。

必须通过：

- UI 尺寸有审批记录或与调用位置一致。
- 文字、图标、边框、圆角、按钮纹理清晰。
- 没有拉伸导致的圆角变形、线框变粗变细或按钮比例异常。
- alpha 通道完整；边缘没有黑边、白边、锯齿或半透明脏边。
- 放大后不出现过度锐化、噪点块、纹理漂移或压缩感。

## 缩略图

缩略图应从最终背景 / CG 重新生成，不直接把低清缩略图放大为最终缩略图。

必须通过：

- 缩略图构图保留关键人物、脸部、道具和画面中心。
- 不模糊、不锯齿、不偏色。
- 与对应最终背景 / CG 内容一致。
- 尺寸符合系统窗口需求或 UI 负责人审批记录。

## 最终交接字段

每张图的 QA 记录至少包含：

- 源路径
- 最终图路径
- 当前状态
- 检查人
- 检查日期
- 通过 / 返工结论
- 返工原因
- 交给谁处理
- 问题截图路径；如文件损坏或为空，写明无法截图并提供问题文件路径
