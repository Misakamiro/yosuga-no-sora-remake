# 1920x1080 UI 和图片尺寸需求

## 共同目标

- 逻辑画布：`1920x1080`。
- 中心点：`960,540`。
- 宽高比：16:9。
- 旧 `800x600` 坐标不能机械放大，除非对应素材也同步放大并通过视觉确认。

## UI 组需求

### 对话框

负责文件：`system/MessageFrame.tjs`

需要：

- 新的 `1920x1080` 对话框背景。
- 新的名字区域和正文文字区域。
- 新的系统菜单条或按钮组。
- 新的语音、关闭、隐藏、闪烁/下一步提示、路线/消息标记位置。
- 与可见按钮一致的点击矩形。

当前需要替换的旧锚点：

- `FRAME_WIDTH = 750`、`FRAME_HEIGHT = 135`。
- `BASE_X = 25`、由底部推导的 `BASE_Y`。
- 名字区域当前为 `800x60`。
- 正文区域当前约 `700` 宽。
- 系统菜单层当前为 `800x33`。
- 隐藏、关闭、闪烁提示位置包含 `747`、`1243`、`1249` 等旧或混合坐标。

推荐方向：

- 底部对话框可以跨更宽区域，也可以使用有意设计的居中宽度。
- 文本宽度按阅读体验设计，不建议默认占满 1920。
- 每个按钮素材交付时必须同时给出点击区域。

### 选项按钮和路线提示

负责文件：`system/ADVScreen.tjs`

需要：

- 适合 `1920x1080` 的选项按钮素材。
- 选项围绕 `WINDOW_CENTER_Y = 540` 排列的垂直中心策略。
- 路线提示图标相对新按钮素材的位置。

当前旧锚点：

- 选项列表垂直中心使用 `300`。
- 路线提示图标使用旧按钮内部位置，例如接近 `x=740`。

### 设置界面

负责文件：`system/ConfigWindow.tjs`

需要：

- 完整 `1920x1080` 设置背景/底层布局。
- 16:9 画面尺寸预设的替换文案。
- 所有标签页、滑条、按钮的新位置。
- 提示弹窗在 `1920x1080` 下的最大宽度和位置测试。

当前旧/部分 HD 锚点：

- `BASE_X = 136`、`BASE_Y = 90`。
- 全局设置底图使用 `1280x720`。
- 画面尺寸选项仍是 4:3：`1600x1200`、`1200x900`、`800x600`、`600x450`、`400x300`。

推荐画面尺寸预设：

- `1920x1080` 原生。
- `1600x900`。
- `1280x720`。
- 可选 `960x540`，用于小窗口。
- 旧 4:3 预设仅在需要兼容时保留，并标注为旧版模式。

### 存档、读档、历史

负责文件：`system/SystemWindow.tjs`

需要：

- `1920x1080` 存档/读档背景。
- 新的存档格布局和缩略图目标尺寸。
- 历史背景、文字栏、滚动条、跳转按钮、预览位置。
- 明确缩略图策略：
  - 保存完整 `1920x1080` 截图并在 UI 内缩小；或
  - 保存固定尺寸缩略图，例如 `320x180` / `480x270`。

当前敏感点：

- 叠层跟随 `WINDOW_WIDTH`、`WINDOW_HEIGHT`。
- 历史预览宽度使用 `1280 * (h / WINDOW_HEIGHT)`。
- 历史消息边距使用 `WINDOW_WIDTH-100`，在 1080p 下可能过宽。

### Album / CG 浏览器

负责文件：`system/Album.tjs`

需要：

- `1920x1080` Album 模式背景和菜单。
- CG 缩略图网格目标尺寸和间距。
- 底部查看菜单高度和点击区域。
- 横竖滑条预览素材和视口裁切矩形。
- 旧 4:3 CG 在 16:9 画布中的平移/裁切交互决策。

当前敏感点：

- 底部查看菜单高度 `306`，底部点击区域 `156`。
- 缩略图网格仍使用旧固定坐标。
- 横竖滑条裁切数学会比较图片尺寸和 `WINDOW_WIDTH`、`WINDOW_HEIGHT`。

### 标题 / Logo

负责文件：`system/Title.tjs`

需要：

- `1920x1080` 标题背景。
- 新标题菜单条和按钮位置。
- 新 Logo/注意事项精灵位置和中心点。
- Bonus 菜单位置。

当前旧锚点：

- 注意事项精灵位置围绕 `x=400`。
- 按钮底层是 `800x90` 并贴底。

## 图片组需求

### 背景和 CG

每张背景/CG 需要决定：

- 重制为 `1920x1080`。
- 从 `800x600` 放大。
- 左右留边。
- 裁切。
- 放入设计框。

交付内容：

- 素材尺寸。
- 预期中心点。
- 如果大于 `1920x1080`，给出安全裁切区域。
- 脚本侧 `center` 元数据是否保持旧值或更新。

### 视频素材

负责文件：`system/Movie.tjs`

需要：

- 确认源视频尺寸。
- 如果视频仍是 `800x600`，给出显示策略：源尺寸居中、整数放大、适配高度、适配宽度或加框。
- 如果视频已重制，提供 `1920x1080` 文件并确认是否存在脚本侧例外。

### UI 框架素材

需要：

- 对话框图片。
- 设置背景和控件。
- 存档/读档/历史背景和控件。
- Album 控件和缩略图。
- 标题、Logo、菜单素材。
- 如继续使用电影遮幅，也要提供对应遮幅素材。

导出要求：

- 最终像素尺寸。
- 在 `1920x1080` 画布上的左上角位置。
- 每个可点击控件的点击矩形。
- 按钮精灵的状态帧数量。
- 每个动态文本区域的文本矩形。

## UI/图片组交接格式

每个素材或控件按以下格式提交：

```text
screen: MessageFrame
asset: FRM_xxxx replacement name
canvas: 1920x1080
asset_size: width x height
position: x,y
center: x,y or none
hit_rect: left,top,right,bottom
text_rect: left,top,right,bottom or none
states: normal/hover/disabled/etc.
notes: any scaling/crop rule
```

## 资源交付验收标准

- 每个可见控件都有对应点击矩形。
- 每个文本区域都有边界矩形和推荐字号。
- 每个旧 `800x600` 中心点都有 `1920x1080` 替换值，或明确记录继续保留旧值的原因。
- 视频和 CG 策略在代码改动播放或 Album 裁切前已明确。
