# 1920x1080 UI 布局标准

## 坐标系统

- 目标画布：`1920x1080`。
- 安全区：重要控件保持在 `x=80..1840`、`y=60..1020`。
- 画面主体保护：除系统独占页面外，避免把不透明面板放在角色主体区域 `x=520..1400`、`y=120..780`。
- 通用边距：桌面基准 `64px`；全屏系统页可用 `80px`。

## 字体

优先使用现有 `font/*` 字体资源，除非字体组提供已验证的 KRKRZ 兼容 `.tft`。

推荐字号：

- 对话正文：`34px`，行高 `46px`。
- 角色名：`30px`。
- 系统正文：`28px`。
- 次级标签和元数据：`22-24px`。
- 页面标题：`48-56px`。
- 按钮文字：`28-32px`。

除非旧代码必须使用位图文字，不建议把文本烘焙到装饰面板中。脚本允许时，优先使用可复用按钮背景加运行时文字。

## 色彩和材质

- 基础面板：半透明近黑或深蓝灰，透明度约 `70-84%`。
- 强调色：柔和青色/天蓝用于焦点和选中；暖白用于文字；琥珀色只用于警告或新标记。
- 避免整页单一高饱和蓝紫。标题可以更温暖，图库更中性，让原作画面成为重点。
- 使用细 1px 高光和 1px 内阴影增强现代感，避免厚重浮雕。

## 交互状态

可复用控件应按需要包含：

- 正常
- 悬停/焦点
- 按下
- 禁用
- 选中/开启
- 锁定或不可用

键盘/手柄焦点不能只依赖颜色，需要增加描边、发光、下划线或标记。

## 页面布局标准

### 标题

- 菜单栈：右侧或右下。
- 主按钮尺寸：`300x58`。
- 垂直间距：`12-16`。
- Bonus 菜单可用较小 `260x52` 按钮，但仍要支持键盘导航。

### 对话

- 面板：`1560x210`，底部居中，`x=180`、`y=810`。
- 文本内区：`1360x130`，建议从 `x=300`、`y=882` 附近开始。
- 名字牌：`300-360x52`，贴在面板左上。
- 控件组：面板右侧，图标点击目标 `52x52`。

### 快捷菜单

- 工具条按钮点击目标：至少 `56x48`。
- 子菜单卡片：最大宽度 `520`，向上或向内展开，不能超出安全区。
- 快速存档卡：缩略图至少 `160x90`，时间戳可读。

### 选项

- 选项按钮：宽 `760-920`，高 `72`。
- 堆叠中心：`x=960`；第一项约 `y=420`。
- 间距：`18`。
- 长文本：先允许两行，再考虑降低字号。

### 设置

- 窗口：`1440x860`，`x=240`、`y=110`。
- 左栏：`260`。
- 内容宽度：`1080`。
- 行高：`54-64`。
- 滑条长度：`420-520`。
- 底部操作行高度：`88`。

### 存档 / 读档

- 窗口：`1600x900`，`x=160`、`y=90`。
- 左栏：`220`。
- 卡片：目标 `300x250`。
- 缩略图：`260x146` 或更大，16:9。
- 卡片元数据：日期、槽位编号、路线/备注、锁定/新标记。

### 历史

- 窗口：`1500x900`，`x=210`、`y=90`。
- 文本列：`1220`。
- 右侧工具栏：`120`。
- 日志行内边距：`24-28`。
- 语音/跳转控件：`48x48`。

### CG 模式

- 左侧分类栏：`280`。
- 主网格：优先 `4x3`，卡片约 `330x210`；备选 `5x2`，卡片约 `292x190`。
- 查看工具条：右下自动淡出，避免覆盖关键画面。
- 锁定 CG 占位：与解锁缩略图同尺寸，低透明度，锁定标记清楚。

### 确认弹窗

- 模态框：`720x260`，居中于 `x=600`、`y=410`。
- 按钮：`180x56`。
- 按钮间距：`20`。
- 复选框行：高 `44`。
- 提示条：紧凑 `720x72`，按调用场景放在顶部或底部附近。

## 素材导出规则

- 最终运行时资源导出为项目期望格式，通常为 `.tlg`。
- 源设计文件由图片组单独保存，除非明确要求，不放入运行时目录。
- 素材图集必须保持代码期望的帧顺序：正常、焦点、禁用、选中/开启、按下等。
- 如果替换现有多帧位图，必须记录帧数和单帧尺寸。
